## What is AI?
The term AI covers a number of software capabilities.
The most well known capability currently is generative AI.
Generative AI solutions involve large language models.Generative AI generates responses to human prompts.For example, chat GPT.
Generative AI agents are applications that will respond to prompts, but can also autonomously act on inputs. For example, an executive assistant agent may be able to book hotels based on a calendar entry.
Computer vision is the ability to interpret images. Instead of predicting the next token as a word, a computer vision AI solution will be able to provide a description to a input based on pixels such as a JPG file.
speech capability involves interpreting a human's speech into text or generating speech from text input.
natural language processing AI can take an input from for example from a book and analyze it, identify key components, key points, natural language processing AI may also be able to interpret the sentiment of the input. AI may also be able to provide decision support by looking at historical examples of information and determining and predicting possible outcomes.

Generative AI solutions are commonly termed as large language models. They predict tokens based on training. Data is fed in and the large language model And then learns which tokens are most likely to occur next based on weights and biases Also known as parameters.

## Microsoft Foundry 

If you want access to AI capabilities through Microsoft Azure, you have two options.
The first is using is creating a Microsoft Foundry project inside which are deployed Microsoft resources such as Azure OpenAI, Azure Vision, Azure Speech.
Interestingly, Azure OpenAI is not available as a standalone resource. It must be deployed within a Microsoft Foundry project.
Not all regions have AI capability. For example, Australia East has foundry capability but other Australia regions do not.

Microsoft provides a large number of AI models. These can be broken down into two main groups. Firstly, open AI models, and secondly, other models. OpenAMI models are for example GPT five point threeAnd GPT five point three codex. Examples of other models include Deep Seek, Llama, and Grok.
